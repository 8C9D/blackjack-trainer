# Security Sanity Check Report

_Generated by the security-sanity-check skill. Originally run 2026-05-28;
re-run and rewritten on 2026-06-08 (branch `main`)._

## 1. Scope

Practical security hygiene sanity check of the `blackjack-trainer` repository
on branch `main`. This is **not** a penetration test or a full audit. It
covers: tracked secrets/sensitive files, `.gitignore` publish safety,
auth/session handling, input validation and unsafe operations, frontend XSS
surface, the native iOS target, dependency hygiene, and CI/CD config.
Inspection was local-only; no external services were contacted beyond
`npm audit`'s advisory lookup.

This run re-verified all prior findings against the current `main`, which has
grown substantially since the last check (now **389 tracked files**, including
a full native iOS/SwiftUI target under `ios/`). One thing changed materially:
`npm audit` now reports **4 moderate dev-only vulnerabilities** (was 0). They
are documented as Finding 6. Everything else still holds.

## 2. Project Overview

- **Web app**: Angular 21 (standalone components, signals), **frontend-only**
  single-page app. TypeScript 5.9 (strict), SCSS, Vitest 4 + jsdom for tests.
- **iOS app**: a native SwiftUI target under `ios/` (`BlackjackTrainer`) that
  re-implements the same training engines. Uses `UserDefaults` and
  `NSUbiquitousKeyValueStore` (iCloud key-value) for per-trainer stats; no
  `WKWebView`, no network requests.
- **Backend**: none. No server, no API layer.
- **Network**: none — no `HttpClient`, `fetch`, `XMLHttpRequest`, `WebSocket`,
  `EventSource`, or `sendBeacon` usage anywhere in `src`; no `URLSession`
  network calls in `ios/`.
- **Persistence**: browser `localStorage` (web) and `UserDefaults` + iCloud KV
  (iOS) — local/device-scoped per-trainer session stats only.
- **Auth / sessions / payments / database**: none.
- **Env vars**: none — no `process.env` / `import.meta.env` usage and no
  Angular `src/environments/` folder.
- **Dependency manager**: npm (`package-lock.json` present).
- **CI/CD**: two GitHub Actions workflows. `.github/workflows/ci.yml` runs
  `npm ci → lint → test → build` plus a parity-fixture anti-drift gate on
  push/PR to `main`. `.github/workflows/ios-ci.yml` runs SwiftFormat/SwiftLint
  and `xcodebuild build test` for `ios/**` changes. Neither embeds secrets.
- **Deploy**: produces a static bundle in `dist/` (`ng build`); iOS builds via
  Xcode.
- **Size**: 389 tracked files.

## 3. Executive Summary

**Overall risk: Low.**

A static, frontend-only web trainer plus a native iOS port, with no backend, no
network calls, no secrets, no authentication, and no user data leaving the
device. No real secrets are tracked or present on disk. `.gitignore` protects
`.env*`, build output, IDE files, and per-machine `.claude/` settings. The one
change since the last clean run is **4 moderate `npm audit` advisories**, all
in a transitive **dev-only** dependency chain (`@angular/cli` → MCP SDK →
`hono`) that never ships in the built bundle and has **no fix available**
(Finding 6). No manual security action is required and no auto-fixes were
applied this run (dependency upgrades are out of scope for auto-fix and, here,
unavailable).

## 4. Findings — web/repo

### Finding 1 — `.gitignore` `.env*` guard (in place, confirmed)

- **Severity**: Low (resolved)
- **Location**: `.gitignore`
- **Evidence**: `.gitignore` contains an "Environment variables / local
  secrets" block with `.env`, `.env.*`, and `!.env.example`. Re-validated:
  `git check-ignore .env .env.local` lists them as ignored while
  `.env.example` remains trackable. Landed in commit `e27dbe5`.
- **Why it matters**: Prevents accidental commits of a future `.env`.
- **Auto-fix status**: Fixed in the 2026-05-28 run; no action this run.
- **Secret redacted**: No (no secret involved).
- **Confidence**: High.

### Finding 2 — No tracked secrets or sensitive files

- **Severity**: Info
- **Location**: whole repo (389 tracked files, including `ios/`).
- **Evidence**: No `.env*`, key, cert, keystore, dump, `.sqlite`/`.db`, or
  service-account files are tracked or present on disk. A keyword scan
  (`API_KEY|SECRET|PASSWORD|PRIVATE_KEY|TOKEN|DATABASE_URL` plus `-----BEGIN`)
  over `src`, `ios`, and `tools` produced no credential matches — the only
  `http://` hits in `ios/` are XML/DTD namespace URLs in SVG assets and the
  entitlements plist DOCTYPE, none of which configure network behavior.
- **Why it matters**: Confirms nothing sensitive is committed.
- **Auto-fix status**: N/A (nothing to fix).
- **Secret redacted**: No real secret found; nothing to redact.
- **Confidence**: High.

### Finding 3 — No unsafe DOM / HTML rendering (XSS surface)

- **Severity**: Info
- **Location**: `src/`
- **Evidence**: Zero occurrences of `innerHTML`, `outerHTML`,
  `insertAdjacentHTML`, `document.write`, `eval(`, `new Function`, or
  Angular's `bypassSecurityTrust*` / `DomSanitizer`. Default contextual
  auto-escaping is intact and unbypassed. The iOS app uses no `WKWebView` /
  `loadHTMLString`, so there is no native HTML-injection sink either.
- **Why it matters**: No injection path from data into the DOM.
- **Auto-fix status**: N/A.
- **Confidence**: High.

### Finding 4 — Persisted stats parsing is type-guarded

- **Severity**: Info
- **Location**: `src/app/core/services/stats-store.ts`; `ios/.../Stores/`
- **Evidence**: The web `StatsStore` wraps `JSON.parse` in `try/catch`, accepts
  a payload only after verifying its numeric fields, copies them into a fresh
  object (no spread of untrusted keys), and falls back to empty stats on
  malformed input; `persist()` tolerates quota/private-mode throws. The iOS
  `StatsStore` decodes from `UserDefaults`/iCloud KV with `Codable` and falls
  back to defaults. Worst case is a stats reset.
- **Why it matters**: No prototype-pollution or crash surface from tampered
  storage.
- **Auto-fix status**: N/A (already safe).
- **Confidence**: High.

### Finding 5 — Application dependencies are clean; no install scripts

- **Severity**: Info
- **Location**: `package.json` / `package-lock.json`
- **Evidence**: Runtime dependencies are Angular 21 framework packages plus
  `rxjs`/`tslib`. `package.json` scripts are plain Angular CLI / tooling
  wrappers with no `postinstall`/`preinstall`/`prepare` lifecycle hooks that
  run arbitrary code on install.
- **Auto-fix status**: N/A.
- **Confidence**: High.

### Finding 6 — 4 moderate `npm audit` advisories in a dev-only chain

- **Severity**: Low (dev-only; not shipped)
- **Location**: transitive devDependencies of `@angular/cli`
- **Evidence**: `npm audit` reports **4 moderate** advisories, all in
  `@angular/cli` → `@modelcontextprotocol/sdk` → `@hono/node-server` / `hono`
  (e.g. "Hono: IP Restriction bypasses static deny rules for non-canonical
  IPv6", "Cookie helper does not sanitize sameSite/priority — Set-Cookie
  injection"). These belong to the optional Angular CLI MCP dev server. **No
  fix is available** for the `@angular/cli` path.
- **Why it matters**: Low impact here — `hono` is a server framework that is
  **never part of the built web bundle or the iOS app**, and the advisories
  concern server request handling the project does not run in production. The
  exposure is limited to a developer's machine if they explicitly launch the
  CLI MCP server.
- **Auto-fix status**: **Skipped (Manual / monitor).** Dependency upgrades are
  out of scope for automatic fixes, and no compatible fix exists yet.
- **Secret redacted**: No.
- **Confidence**: High.

## 5. Secrets and Sensitive Files Review

No `.env` files (tracked or on disk). No private keys, certificates,
keystores, database dumps, or cloud service-account files in `src`, `ios`, or
`tools`. The iOS `.entitlements` file declares only an iCloud key-value-store
identifier built from Xcode build variables (`$(TeamIdentifierPrefix)$(...)`) —
no literal team ID or credential. `.claude/` and `dist/` exist on disk but are
correctly untracked (verified via `git check-ignore`). No secret values are
reproduced in this report.

## 6. `.gitignore` and Publish Safety Review

`.gitignore` covers `dist`, `tmp`, `out-tsc`, `bazel-out`, `node_modules`,
debug logs, `.angular/cache`, `coverage`, OS files, common IDE files,
`.claude/`, iOS build artifacts (`ios/build/`, `xcuserdata`, `*.xcuserstate`),
and several intentionally-untracked scratch docs. The `.env*` guard with the
`!.env.example` negation is present and validated. A `.vscode/*` allowlist
tracks only `settings/tasks/launch/extensions/mcp.json`; `mcp.json` merely
launches the official `@angular/cli mcp` server via `npx` with no embedded
credentials.

## 7. Authentication and Authorization Review

**Not applicable.** No authentication, authorization, sessions, cookies, or
user accounts in either target. No server to enforce anything. No hardcoded
users, no plaintext password storage, no admin routes.

## 8. Input Validation and Unsafe Operation Review

No SQL (no database), no shell execution, no filesystem access, no
deserialization of untrusted input beyond the type-guarded persisted-stats
reads in Finding 4, and no network input. User input is limited to numeric
drill answers and settings, consumed in-memory by pure engines (TypeScript /
Swift). No open-redirect surface and no unsafe HTML rendering (Finding 3).

## 9. Dependency and Tooling Review

`npm audit` → **4 moderate** advisories, all dev-only in the `@angular/cli`
MCP-server → `hono` chain with no fix available (Finding 6). Application
runtime dependencies (Angular framework, `rxjs`, `tslib`) carry no advisories.
No install-time lifecycle scripts. iOS lint/build tooling (SwiftFormat,
SwiftLint, xcodebuild) is invoked only in CI.

## 10. CI/CD and Deployment Review

Two GitHub Actions workflows, neither embedding credentials:

- `.github/workflows/ci.yml` — on push/PR to `main`: `npm ci → npm run lint →
  CI=true npm test → npm run build`, plus an anti-drift gate that regenerates
  parity fixtures and fails on any diff. Uses pinned official actions
  (`actions/checkout@v5`, `actions/setup-node@v5`).
- `.github/workflows/ios-ci.yml` — on `ios/**` changes: `brew install
  swiftformat swiftlint`, lint gates, then `xcodebuild build test` on a
  macOS runner.

No secrets, tokens, or deploy credentials are referenced in either workflow.
Deployment artifact is a static `dist/` bundle (gitignored).
