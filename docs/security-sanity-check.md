## 1. Scope

Practical security hygiene sanity check of the `blackjack-trainer` repository
on branch `main`. This is **not** a penetration test or a full audit. It
covers: tracked secrets/sensitive files, `.gitignore` publish safety,
auth/session handling, input validation and unsafe operations, frontend XSS
surface, dependency hygiene, and CI/CD config. Inspection was local-only; no
external services were contacted beyond `npm audit`'s advisory lookup.

This run re-verified all findings from the 2026-05-28 check against the
current `main` (which has since absorbed the earlier cleanup branch and
several refactor commits). The single fix from that run — the `.env*`
`.gitignore` guard — is now part of `main` (commit `e27dbe5`) and was
re-validated. No new hygiene gaps were found and no new fixes were needed.

## 2. Project Overview

- **Type**: Angular 21 (standalone components, signals), **frontend-only**

- **Backend**: none. No server, no API layer.
- **Network**: none — no `HttpClient`, `fetch`, `XMLHttpRequest`, `axios`,
  `WebSocket`, `EventSource`, or `sendBeacon` usage anywhere in `src`.
- **Persistence**: browser `localStorage` only (per-trainer session stats).
- **Auth / sessions / payments / database**: none.
- **Env vars**: none — no `process.env` / `import.meta.env` usage and no
  Angular `src/environments/` folder.
- **Dependency manager**: npm (`package-lock.json` present).
- **CI/CD**: none in-repo (`.github/workflows/` and other pipeline files
  absent).
- **Deploy**: produces a static bundle in `dist/` (`ng build`).
- **Size**: 151 tracked files.

## 4. Findings

### Finding 1 — `.gitignore` `.env*` guard (previously fixed, now confirmed)

- **Severity**: Low (resolved)
- **Location**: `.gitignore`
- **Evidence**: `.gitignore` now contains an "Environment variables / local
  secrets" block with `.env`, `.env.*`, and `!.env.example`. Re-validated:
  `git check-ignore .env .env.local .env.production` lists all three as
  ignored, while `.env.example` remains trackable. The fix landed in commit
  `e27dbe5` ("chore: improve security hygiene") and is present on `main`.
- **Why it matters**: If a contributor later introduces a `.env` with API
  keys or other secrets (common when a backend is added), the ignore rule
  prevents accidental commits. Publish-safety hygiene.
- **Auto-fix status**: **Fixed in the 2026-05-28 run; no action this run.**
- **Secret redacted**: No (no secret involved).
- **Confidence**: High.

### Finding 3 — No unsafe DOM / HTML rendering (XSS surface)

- **Severity**: Info
- **Location**: `src/`

- **Why it matters**: No injection path from data into the DOM.
- **Auto-fix status**: N/A.
- **Confidence**: High.

### Finding 4 — `localStorage` parsing is type-guarded

- **Severity**: Info
- **Location**: `src/app/core/services/stats-store.ts`
- **Evidence**: The store was refactored since the last check into a
  `StatsStore` class parameterized by a storage key (multiple feature areas
  persist independent stats). The read path is unchanged in substance:
  `load()` wraps `JSON.parse` in `try/catch` and only accepts the value after
  verifying all four fields (`attempts`, `correct`, `streak`,
  `longestStreak`) are `number`s, copying them explicitly into a fresh object
  (no spread of untrusted keys). Malformed or missing payloads fall back to
  `EMPTY_STATS`. `persist()` tolerates quota/private-mode throws. The new
  `cleanupLegacyStatsKeys()` only calls `localStorage.removeItem()` on a
  hardcoded allowlist of old keys — it parses nothing.
- **Why it matters**: No prototype-pollution or crash surface from tampered
  `localStorage`; worst case is a stats reset, which is acceptable.
- **Auto-fix status**: N/A (already safe).
- **Confidence**: High.

`.gitignore` covers `dist`, `tmp`, `out-tsc`, `bazel-out`, `node_modules`,

## 7. Authentication and Authorization Review

**Not applicable.** The app has no authentication, authorization, sessions,

## 8. Input Validation and Unsafe Operation Review
